ABSTRACT
================================================================

The Health & Meal Management System project is a mobile application developed using
the Flutter framework and Firebase cloud services. The objective of this project is to
design and implement a user-friendly application that helps individuals track their daily
meals, manage dietary habits, and maintain healthier lifestyles. The application allows
users to record, update, and monitor their meal information while securely storing the
data using Firebase Cloud Firestore. This project focuses on implementing the core
functionalities required for efficient meal management, including user authentication,
meal logging, meal editing, and real-time data synchronization.

The system ensures secure and reliable access through Firebase Authentication, allowing
users to safely manage their personal meal records. The application also supports features
such as meal categorization, profile management, and viewing meal history to help users
maintain consistent dietary tracking. These features are designed to provide a seamless
and intuitive experience for users who wish to monitor their nutritional intake and
maintain a structured record of their daily food consumption.

In addition to individual meal management, the system incorporates group collaboration
features where users can create groups, send invitations, and manage group participation.
This functionality encourages accountability and motivation by allowing users to track
their progress together with friends or family members. The group-based tracking system
is designed to foster a sense of community and shared responsibility among users who
are working towards common health and dietary goals.

The application ensures smooth operation by implementing structured data management,
responsive design, and efficient communication between the mobile interface and cloud
database. The system is built using Flutter 3.x and Firebase SDK, targeting both Android
and iOS platforms, ensuring broad accessibility and cross-platform compatibility.

The main goal of this project is to demonstrate the practical implementation of mobile
application development, cloud-based database management, and secure authentication
mechanisms using modern technologies such as Flutter and Firebase. The project provides
a structured example of how digital solutions can support healthy lifestyle management
by transforming traditional meal tracking methods into a scalable, cloud-backed mobile
solution that bridges the gap between manual dietary tracking and intelligent health
management.

CHAPTER 1: INTRODUCTION

1.1 Background of the Project
----------------------------------------------------------------

With the increasing awareness of health and nutrition, many individuals are becoming
more conscious about maintaining balanced diets and healthy lifestyles. However, keeping
track of daily meals and dietary habits can be challenging without a proper system. Many
people rely on memory or manual notes to record their meals, which often leads to
inconsistent tracking and difficulty in monitoring eating habits over time. The absence
of a structured and accessible system makes it difficult for individuals to identify
patterns in their dietary behavior and take corrective action when necessary.

With the advancement of mobile technology, digital health applications have become
popular tools for helping individuals manage their health-related activities. Mobile
applications provide convenient ways for users to record data, track progress, and access
information anytime and anywhere. Health-related mobile applications can assist users in
maintaining structured records of their meals and improving their overall dietary habits.
The widespread adoption of smartphones has created an opportunity to develop digital
tools that are both accessible and easy to use for people of all age groups.

The Health & Meal Management System is a mobile application designed to help users
track their daily meals and manage their dietary activities efficiently. The system allows
users to add, update, and view meal records through a simple and user-friendly interface.
The application also integrates group features that allow users to collaborate with others
and stay motivated in maintaining healthy eating habits. By combining individual meal
tracking with social accountability features, the system aims to provide a comprehensive
solution for dietary management.

This project is developed using the Flutter framework for mobile application development
and Firebase for backend services such as authentication and real-time database management.
Flutter, developed by Google, enables the creation of natively compiled applications for
mobile, web, and desktop from a single codebase, making it an ideal choice for this
project. Firebase provides a robust and scalable backend infrastructure that supports
real-time data synchronization, secure authentication, and cloud storage.

The system focuses on implementing essential features including meal tracking, secure
user login, group management, and responsive user interface design to provide a practical
and efficient digital solution for meal management. The application is designed to be
intuitive and accessible, ensuring that users with varying levels of technical expertise
can effectively use the system to manage their dietary habits and work towards their
health goals.

1.2 Problem Statement
----------------------------------------------------------------

Maintaining healthy eating habits requires consistent monitoring of daily meals and
dietary activities. Many individuals find it difficult to track their meals regularly due
to lack of proper tools or structured systems. Traditional methods such as manual notes
or basic tracking applications are often inefficient and may not provide real-time access
to stored information. These limitations make it challenging for users to maintain
consistent records and gain meaningful insights from their dietary data.

Existing meal tracking systems may provide limited functionality, focusing only on basic
meal logging without offering features such as secure authentication, group collaboration,
and real-time data synchronization. Popular applications such as generic calorie counters
often lack real-time collaborative features and require paid subscriptions for cloud sync,
making them inaccessible for everyday users. In addition, some systems store information
locally, which may lead to data loss or limited accessibility across devices.

The lack of collaborative features in existing systems is a significant limitation, as
social support and accountability play an important role in maintaining healthy lifestyle
habits. Without the ability to share progress and collaborate with others, users may lose
motivation to continue tracking their meals consistently. This highlights the need for a
system that combines individual meal tracking with group-based collaboration features.

With the increasing use of smartphones and cloud-based technologies, there is a need for
a digital system that allows users to manage their meals efficiently while ensuring secure
data storage and accessibility. The system should provide an organized platform for
recording meals, managing user profiles, and collaborating with other users to encourage
healthy lifestyle habits.

This project addresses these challenges by developing a Health & Meal Management System
that allows users to track their meals, store data securely using cloud services, and
collaborate with others through group features. The system aims to provide a reliable and
user-friendly mobile application for improving meal tracking and promoting healthy eating
habits among individuals who are committed to maintaining a balanced and nutritious diet.

1.3 Objectives of the Project
----------------------------------------------------------------

The main objective of this project is to design and develop a Health & Meal Management
System using Flutter and Firebase that allows users to track their meals and maintain
healthy dietary habits through a digital platform. The specific objectives of the project
are as follows:

  • To develop a mobile application that allows users to record and manage their daily
    meals, including breakfast, lunch, dinner, and snacks.

  • To implement secure user authentication using Firebase Authentication for safe and
    reliable access to the system.

  • To store meal data using Firebase Cloud Firestore with real-time synchronization,
    ensuring data is always up to date across all devices.

  • To implement meal management features such as adding, editing, deleting, and viewing
    meal entries with detailed descriptions and timestamps.

  • To develop group management functionality where users can create groups, invite
    members, and participate in collaborative health tracking activities.

  • To design a responsive and user-friendly interface using Flutter widgets for easy
    interaction with the system across different screen sizes and devices.

  • To ensure smooth application workflow and reliable data management using Firebase
    cloud services, minimizing latency and data inconsistencies.

  • To demonstrate the use of modern mobile application development technologies such as
    Flutter and Firebase in building a practical health management solution.

  • To conduct thorough testing and debugging to ensure the application operates correctly
    and meets the functional requirements defined during the planning phase.

1.4 Scope and Limitations
----------------------------------------------------------------

1.4.1 Scope of the Project

The scope of the Health & Meal Management System defines the boundaries and extent of
the features and functionalities that are implemented in the current version of the
application. The following points describe the scope of the project:

  • The project focuses on developing a mobile application for managing daily meal records,
    providing users with a structured and accessible platform for dietary tracking.

  • The system allows users to track meals such as breakfast, lunch, dinner, and snacks,
    with the ability to add descriptions and timestamps to each meal entry.

  • The application provides secure user authentication and cloud-based data storage using
    Firebase services, ensuring that user data is protected and accessible from any device.

  • The system allows users to create groups and invite other users for collaborative
    tracking, encouraging shared participation in health management activities.

  • The application provides a responsive user interface suitable for smartphones and
    tablets, ensuring consistent usability across different screen sizes and resolutions.

  • The project demonstrates practical implementation of mobile application development
    using Flutter and Firebase, serving as a structured academic project that showcases
    modern development techniques.

  • The system supports real-time data synchronization, ensuring that any updates made
    by users are immediately reflected across all connected devices and sessions.

1.4.2 Limitations of the Project

While the Health & Meal Management System provides a comprehensive set of features for
meal tracking and group collaboration, there are certain limitations in the current
version of the application:

  • The system currently focuses only on meal tracking and does not include detailed
    calorie analysis or nutritional calculations. Users cannot view macronutrient breakdowns
    or caloric values for their recorded meals.

  • The application does not include integration with wearable health devices such as
    fitness trackers or smartwatches, which could provide additional health data.

  • The system is developed as an academic project, so large-scale commercial deployment
    is not considered. The application has not been optimized for handling a very large
    number of concurrent users.

  • Advanced features such as AI-based diet recommendations and personalized meal planning
    are not implemented in the current version of the application.

  • The application has been tested primarily on Android devices. Full iOS compatibility
    testing is pending and will be addressed in the next phase of development.

  • The system does not currently support offline functionality. Users require an active
    internet connection to access and update their meal records.

CHAPTER 2: LITERATURE REVIEW
================================================================

This chapter reviews existing research and technologies related to mobile health
applications, meal tracking systems, cloud-based data management, and modern mobile
application development frameworks. The purpose of this review is to understand how
digital health systems help users track their daily activities and maintain healthy
lifestyles using mobile technology. It also examines the technologies and design
principles that support the development of such systems.

With the rapid growth of smartphone usage and mobile internet connectivity, mobile
applications have become important tools for managing personal health information. Many
researchers and developers have focused on creating digital solutions that assist users
in monitoring their daily routines, dietary habits, and overall health conditions. These
applications provide convenient and accessible ways for users to record information,
analyze patterns, and make improvements to their lifestyle habits. The growing body of
research in this area highlights the significant potential of mobile technology in
transforming personal health management.

2.1 Mobile Health Applications
----------------------------------------------------------------

Mobile health applications, commonly referred to as mHealth applications, are widely
used for monitoring personal health activities such as physical exercise, diet management,
sleep tracking, and daily activity monitoring. These applications allow users to collect
and store health-related data through mobile devices, making health management more
convenient and accessible. The proliferation of smartphones and mobile internet has
created a large and growing market for health-related applications that cater to a wide
range of user needs and preferences.

According to Free et al. (2013), mobile health technologies play an important role in
improving healthcare delivery and encouraging healthy lifestyle habits. Mobile applications
provide interactive tools that allow users to record their health data, monitor progress,
and receive feedback on their daily activities. Many health applications also include
features such as reminders, progress tracking, and goal setting to motivate users to
maintain consistent healthy habits.

The increasing availability of smartphones and mobile applications has made it easier
for individuals to manage their personal health information effectively. Research has
shown that users who regularly engage with health tracking applications are more likely
to maintain consistent dietary habits and achieve their health goals compared to those
who rely on traditional tracking methods. This underscores the importance of developing
well-designed and feature-rich health management applications that can engage users over
the long term.

Furthermore, the integration of social features in health applications has been shown to
improve user engagement and motivation. Applications that allow users to share their
progress and collaborate with others tend to have higher retention rates and better
health outcomes for their users. This insight has directly influenced the design of the
Health & Meal Management System, which incorporates group collaboration features to
encourage shared participation in health management activities.

2.2 Meal Tracking Systems
----------------------------------------------------------------

Meal tracking systems are designed to help individuals monitor their daily food intake
and maintain balanced diets. These systems allow users to record meals, review dietary
patterns, and track their progress over time. By maintaining consistent records of meals,
users can become more aware of their eating habits and make better nutritional decisions.
The ability to visualize dietary patterns over time is a key feature of effective meal
tracking systems, as it allows users to identify areas for improvement and make informed
changes to their dietary habits.

Research conducted by Direito et al. (2017) suggests that digital meal tracking
applications improve dietary awareness and support long-term behavior change. These
systems help users understand their eating patterns and encourage them to adopt healthier
dietary practices. The study found that users who consistently tracked their meals using
digital applications showed significant improvements in their dietary habits compared to
those who used traditional tracking methods.

However, many existing meal tracking systems focus mainly on simple meal logging and do
not provide additional features such as real-time data synchronization or group
collaboration. As a result, users may lose motivation to continue tracking their meals
consistently. The lack of social features and real-time data access are significant
limitations of many existing meal tracking applications, and addressing these gaps is a
primary motivation for the development of the Health & Meal Management System.

Additionally, many existing meal tracking applications require users to manually search
for and select food items from a database, which can be time-consuming and discouraging.
A simpler approach that allows users to describe their meals in their own words, as
implemented in this project, can lower the barrier to consistent meal tracking and
improve user engagement over time.

2.3 Cloud-Based Data Management
----------------------------------------------------------------

Cloud computing technologies have become essential components of modern mobile
applications. Cloud platforms allow applications to store and manage data securely while
providing access from multiple devices. This ensures that users can retrieve their
information anytime and anywhere without losing their data. The shift towards cloud-based
data management has been driven by the increasing need for scalable, reliable, and
cost-effective storage solutions that can support the growing demands of mobile
applications.

According to Al-Fuqaha et al. (2015), cloud-based systems improve application scalability
and reliability by providing centralized data storage and efficient data synchronization.
These systems also reduce the need for maintaining complex server infrastructure, allowing
developers to focus on building application features rather than managing backend systems.
The use of cloud services has become a standard practice in modern mobile application
development, enabling developers to build scalable and reliable applications with minimal
infrastructure overhead.

Firebase is one of the widely used cloud platforms that provides services such as
authentication, database management, and cloud storage for mobile applications. Using
Firebase Cloud Firestore allows developers to implement real-time data synchronization,
ensuring that updates made by users are instantly reflected in the application. Firestore's
document-based data model provides a flexible and scalable approach to data storage that
is well-suited for the dynamic and evolving data requirements of health management
applications.

The real-time synchronization capabilities of Firebase Cloud Firestore are particularly
valuable for collaborative applications, as they ensure that all users within a group
can see the most up-to-date information without needing to manually refresh the
application. This feature is a key component of the group collaboration functionality
implemented in the Health & Meal Management System.

2.4 Cross-Platform Mobile Application Development
----------------------------------------------------------------

Cross-platform mobile development frameworks enable developers to build applications
that run on multiple operating systems using a single codebase. This approach reduces
development time and ensures consistent application performance across different devices.
The ability to target multiple platforms with a single codebase is a significant
advantage for academic and small-scale projects, as it allows developers to maximize
their reach without duplicating development effort.

Flutter, developed by Google, has become one of the most popular frameworks for building
cross-platform mobile applications. Flutter uses the Dart programming language and
provides a rich set of user interface components that allow developers to create
responsive and visually appealing applications. As of 2023, Flutter has further matured
with improved performance on both Android and iOS, making it an even stronger choice
for cross-platform development (Google Developers, 2023).

According to Nguyen and Park (2020), Flutter offers high performance and flexibility for
mobile application development. Its widget-based architecture allows developers to design
customizable user interfaces and maintain efficient application structure. The hot reload
feature of Flutter significantly speeds up the development process by allowing developers
to see the effects of code changes in real time without restarting the application.

Flutter's widget-based architecture is particularly well-suited for building complex user
interfaces with multiple interactive components. The framework provides a comprehensive
set of pre-built widgets that can be customized and combined to create sophisticated
user interfaces, reducing the amount of custom code required and improving development
efficiency. This has been a key factor in the successful implementation of the responsive
and feature-rich user interface of the Health & Meal Management System.

2.5 Authentication and Security in Mobile Applications
----------------------------------------------------------------

Security is a critical aspect of mobile applications, especially when they handle
personal user data. Authentication mechanisms are used to verify user identity and
prevent unauthorized access to application resources. The importance of robust
authentication systems has grown significantly with the increasing amount of sensitive
personal data stored in mobile applications.

Firebase Authentication provides secure login methods such as email and password
authentication, as well as support for third-party identity providers such as Google
and Facebook. According to Patel and Sharma (2019), implementing secure authentication
systems improves user data protection and increases application reliability. Secure
authentication ensures that users can safely access their accounts and manage their
personal information without the risk of data breaches or unauthorized access.

In addition to authentication, data security in mobile applications involves protecting
data in transit and at rest. Firebase provides built-in security features such as
SSL/TLS encryption for data in transit and configurable security rules for controlling
access to Firestore data. These features ensure that user data is protected at all stages
of its lifecycle, from creation to storage and retrieval.

The implementation of proper security measures is particularly important for health
management applications, as they handle sensitive personal information related to users'
dietary habits and health goals. The Health & Meal Management System addresses these
security requirements by implementing Firebase Authentication for user identity
verification and Firestore security rules for controlling data access.

2.6 User Interface Design in Mobile Applications
----------------------------------------------------------------

User interface design plays a significant role in determining the usability and
effectiveness of a mobile application. A well-designed interface ensures that users can
easily navigate the application and interact with its features. The quality of the user
interface is often a key factor in determining whether users will continue to use an
application over the long term.

According to Tidwell (2020), effective user interface design focuses on simplicity,
clarity, and responsiveness. Mobile applications must provide intuitive navigation and
visually organized layouts to improve the overall user experience. The principles of
good UI design emphasize the importance of consistency, feedback, and error prevention
in creating interfaces that are both functional and enjoyable to use.

Responsive design is particularly important because mobile applications are used on
devices with different screen sizes and resolutions. A responsive interface adapts to
various devices such as smartphones and tablets while maintaining consistent usability.
Flutter's widget-based architecture provides excellent support for responsive design,
allowing developers to create interfaces that automatically adapt to different screen
sizes and orientations.

The design of the Health & Meal Management System's user interface has been guided by
these principles, with a focus on creating a clean, intuitive, and visually appealing
interface that makes it easy for users to record and manage their meals. The use of
Flutter's Material Design widgets ensures a consistent and familiar look and feel that
aligns with users' expectations for modern mobile applications.

2.7 Research Gap
----------------------------------------------------------------

Although many mobile health applications and meal tracking systems exist, several
limitations can be observed in current implementations. Many applications focus mainly
on individual meal logging and lack collaborative features that allow users to interact
with others. In addition, some systems do not provide real-time synchronization or
secure cloud-based data management. Without these features, users may experience delays
in accessing their data or face difficulties when using the application across multiple
devices.

The absence of group collaboration features in most existing meal tracking applications
is a significant gap, as social support and accountability are known to be important
factors in maintaining healthy lifestyle habits. Applications that allow users to share
their progress and collaborate with others tend to have higher engagement rates and
better health outcomes for their users.

Furthermore, most existing systems lack personalized health goal tracking integrated
with meal data, which this project aims to partially address in future phases. The
ability to set and track health goals in conjunction with meal data would provide users
with a more comprehensive view of their progress and help them make more informed
decisions about their dietary habits.

This project addresses these limitations by developing a Health & Meal Management System
using Flutter and Firebase, which integrates secure authentication, real-time database
synchronization, responsive user interface design, and group-based collaboration to
improve meal tracking and encourage healthy lifestyle habits. By combining these features
in a single, cohesive application, the system aims to provide a more complete and
effective solution for dietary management than existing applications.

3.1 Overview of Development Phase
----------------------------------------------------------------

The development phase focuses on implementing the core functionality of the Health &
Meal Management System using the Flutter framework and Firebase cloud services. The main
objective of this phase was to transform the concept of manual meal tracking into a
digital system that allows users to efficiently record and manage their daily meals
through a mobile application. This phase represents the practical realization of the
system design and architecture defined during the planning phase.

During the development process, several key components were implemented including the
authentication system, meal management system, group collaboration module, real-time
database integration, and user interface design. These components were designed as
independent modules in order to maintain a modular and well-organized system architecture.
The modular approach ensures that each component can be developed, tested, and maintained
independently, reducing the risk of introducing errors when making changes to one part
of the system.

A central application controller manages the overall workflow of the system and
coordinates communication between different modules such as user authentication, database
services, and the user interface. This modular design improves system maintainability,
scalability, and simplifies debugging during the development process. The controller
acts as an intermediary between the various modules, ensuring that data flows correctly
through the system and that each module receives the information it needs to function
properly.

The development process began with system planning and application design, followed by
implementation of user authentication and database integration. After establishing secure
user access, the meal management system and group management modules were implemented.
Finally, the user interface and interaction features were integrated and tested to ensure
correct system functionality. This sequential approach ensured that each component was
properly implemented and tested before moving on to the next phase of development.

3.1.1 System Development Approach
----------------------------------------------------------------

A modular development approach was adopted for implementing the Health & Meal Management
System. In this approach, the application is divided into multiple modules, where each
module is responsible for a specific functionality within the system. This approach
promotes separation of concerns and makes the codebase easier to understand, maintain,
and extend.

The major development modules include:

  • Authentication System – Handles user registration and login functionality using
    Firebase Authentication. This module is responsible for verifying user identity and
    managing user sessions throughout the application lifecycle.

  • Meal Management System – Allows users to add, update, delete, and manage meal records.
    This module provides the core functionality of the application and is the primary
    interface through which users interact with their dietary data.

  • Group Management System – Enables users to create groups and invite other members for
    collaborative meal tracking. This module facilitates social interaction and
    accountability among users who share common health goals.

  • Database Management System – Stores and synchronizes user data using Firebase Cloud
    Firestore. This module handles all data persistence and retrieval operations,
    ensuring that user data is always up to date and accessible.

  • User Interface System – Provides visual components and enables user interaction through
    a responsive and intuitive interface built using Flutter widgets. This module is
    responsible for presenting data to users and capturing their input.

By implementing these modules independently, the system becomes easier to maintain and
extend. This approach also allows additional features to be integrated in future versions
without affecting the existing system. The modular architecture also facilitates parallel
development, where different team members can work on different modules simultaneously
without interfering with each other's work.

3.1.2 Development Workflow
----------------------------------------------------------------

The Health & Meal Management System was developed using a structured workflow to ensure
proper system organization and functionality. The development process followed a series
of well-defined steps, each building upon the previous one to create a complete and
functional application.

The development process began with application design and requirement analysis, where
the overall structure and functionality of the system were defined. This phase involved
identifying the key features and functionalities required by the system, defining the
data model, and designing the user interface layouts. The output of this phase was a
detailed specification document that guided the subsequent development activities.

Next, the authentication system was implemented to allow users to securely access the
application. This involved integrating Firebase Authentication into the Flutter
application and implementing the user registration and login screens. The authentication
system was thoroughly tested to ensure that it correctly handles various scenarios such
as invalid credentials, network errors, and session expiration.

After establishing the authentication system, the database integration using Firebase
Cloud Firestore was implemented to store and retrieve user meal data. This involved
defining the Firestore data model, implementing the database service layer, and
integrating it with the application's business logic. The database integration was
designed to support real-time synchronization, ensuring that any changes made by users
are immediately reflected in the application.

Following this, the meal management system was developed to allow users to create, edit,
and view meal records. This module provides the core functionality of the application
and was implemented with a focus on usability and reliability. The meal management
system was tested extensively to ensure that it correctly handles all meal-related
operations and provides a smooth user experience.

The group collaboration module was then implemented to allow users to create groups and
invite members. This module required careful design to ensure that group invitations and
membership changes are handled correctly and that all group members have access to the
latest information. Finally, the user interface design was integrated and tested to
ensure smooth system interaction and proper application functionality.

3.2 System Architecture Design
----------------------------------------------------------------

The Health & Meal Management System follows a client-server architecture, where the
Flutter mobile application acts as the client interface and Firebase provides backend
services such as authentication and database management. In this architecture, different
system components operate independently while communicating through structured data
exchanges. This modular structure reduces system complexity and improves scalability.

[Figure 3.2: System Architecture of Health & Meal Management System]

The system architecture diagram illustrates the interaction between the Flutter client
application and the Firebase backend services. The client application communicates with
Firebase Authentication for user identity verification and with Firebase Cloud Firestore
for data storage and retrieval. The architecture is designed to be scalable and reliable,
ensuring that the system can handle increasing numbers of users and data without
degrading performance.

The primary modules implemented in the system include:

  • Authentication System
  • Meal Management System
  • Group Management System
  • Database Management System
  • User Interface System

These modules work together to manage the application workflow from user login to meal
tracking and group collaboration. The interaction between modules is carefully managed
to ensure data consistency and application reliability.

3.2.1 Authentication System
----------------------------------------------------------------

The Authentication System manages secure user access to the application. This module
uses Firebase Authentication to verify user identity and allow authorized access to the
application features. Firebase Authentication provides a robust and secure authentication
infrastructure that handles the complexities of user identity management, allowing the
application to focus on its core functionality.

The main responsibilities of the authentication system include:

  • User registration – Allows new users to create accounts using their email address
    and a secure password. The registration process includes input validation to ensure
    that users provide valid email addresses and strong passwords.

  • User login – Allows registered users to access the application using their credentials.
    The login process includes error handling to provide informative feedback when users
    enter incorrect credentials.

  • Secure authentication validation – Verifies user credentials against Firebase
    Authentication and manages the authentication state throughout the application lifecycle.

  • User session management – Maintains user sessions across application restarts,
    allowing users to remain logged in until they explicitly log out.

  • Secure access to database services – Ensures that only authenticated users can access
    their personal data and application features, preventing unauthorized access to
    sensitive information.

This system ensures that only authorized users can access their personal data and
application features, providing a secure foundation for the entire application.

3.2.2 Meal Management System
----------------------------------------------------------------

The Meal Management System allows users to record and manage their daily meals within
the application. Each meal entry contains information such as meal type, meal description,
and date. The system is designed to be simple and intuitive, allowing users to quickly
record their meals without requiring extensive input.

Each meal document in Firestore contains the following fields:
  • userId (String)      – Unique identifier of the user who created the meal entry
  • mealType (String)    – Category of meal such as breakfast, lunch, dinner, or snack
  • description (String) – Detailed description of the meal entered by the user
  • timestamp (Timestamp)– Date and time when the meal entry was recorded
  • imageUrl (String)    – Optional field for future photo logging support

The system allows users to:

  • Add new meals – Users can create new meal entries by selecting the meal type,
    entering a description, and submitting the form. The application automatically
    records the timestamp of the meal entry.

  • Edit existing meal entries – Users can modify the details of previously recorded
    meals, allowing them to correct errors or add additional information.

  • Delete meal records – Users can remove meal entries that are no longer needed,
    keeping their meal history clean and organized.

  • View meal history – Users can browse their complete meal history, filtered by date
    or meal type, to review their dietary patterns over time.

Meal data is stored in Firebase Cloud Firestore, allowing real-time synchronization
across devices. This ensures that users can access their meal records from any device
and that any changes are immediately reflected across all connected sessions.

3.2.3 Database Management System
----------------------------------------------------------------

The Database Management System handles the storage and retrieval of application data.
Firebase Cloud Firestore is used as the database system to store information such as
user details, meal records, group data, and notifications. Firestore's document-based
data model provides a flexible and scalable approach to data storage that is well-suited
for the dynamic requirements of the Health & Meal Management System.

Firestore enables real-time synchronization, ensuring that any updates made by users
are immediately reflected within the application. This is achieved through Firestore's
real-time listeners, which automatically notify the application when data changes occur
in the database. This feature is particularly valuable for the group collaboration
functionality, as it ensures that all group members can see the most up-to-date
information without needing to manually refresh the application.

common operations such as retrieving all meals for a specific user or listing all members
of a group.

3.2.4 Group Management System
----------------------------------------------------------------

The Group Management System allows users to collaborate with other users within the
application. Users can create groups and invite other users to participate in
collaborative meal tracking. This feature is designed to foster a sense of community
and shared responsibility among users who are working towards common health goals.

The main functions of this system include:

  • Creating groups – Users can create new groups by providing a group name and
    description. The user who creates the group automatically becomes the group
    administrator.

  • Sending invitations to users – Group administrators can invite other users to join
    their group by entering the invitee's email address. The system sends a notification
    to the invited user, who can then accept or reject the invitation.

  • Accepting or rejecting group invitations – Users can view their pending invitations
    and choose to accept or reject them. Accepted invitations result in the user being
    added to the group's member list.

  • Viewing group members – Users can view the list of members in their groups, along
    with each member's recent meal activity.

This feature encourages accountability and motivation by allowing users to track their
meal activities collectively. The social aspect of group tracking has been shown to
improve user engagement and motivation, making it more likely that users will maintain
consistent meal tracking habits over time.

3.4 Meal Management Implementation
----------------------------------------------------------------

The meal management system is a fundamental component of the Health & Meal Management
System. Each meal entry is represented as a structured data object stored in Firebase
Cloud Firestore. When a user records a meal, the application sends the meal data to the
Firestore database where it is stored and synchronized in real time.

The meal management implementation follows a layered architecture, with a clear
separation between the user interface layer, the business logic layer, and the data
access layer. The user interface layer is responsible for presenting meal data to users
and capturing their input. The business logic layer processes user input and applies
validation rules before passing data to the data access layer. The data access layer
handles all interactions with Firebase Cloud Firestore, including reading and writing
meal data.

The meal entry form includes input validation to ensure that users provide complete and
valid information before submitting a meal record. Validation checks include verifying
that the meal type has been selected, that the description is not empty, and that the
date is valid. If any validation checks fail, the user is presented with informative
error messages that guide them in correcting their input.

The meal history view displays a chronological list of the user's meal records, with
the most recent entries shown first. Users can filter their meal history by date range
or meal type to quickly find specific records. Each meal entry in the history view
displays the meal type, description, and timestamp, providing users with a clear and
organized view of their dietary history.

This system allows users to maintain consistent records of their meals while ensuring
that data remains secure and accessible across devices. The real-time synchronization
provided by Firestore ensures that users can access their meal records from any device
and that any changes are immediately reflected across all connected sessions.
